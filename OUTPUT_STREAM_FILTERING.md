# Output Stream Machine-Specific Filtering

**Date**: January 24, 2026
**Status**: ✅ **COMPLETE AND DEPLOYED**

---

## Overview

Implemented machine-specific filtering for the output stream visualization. Now the output stream in the top-right corner only displays outputs generated by the currently selected machine, filtering out outputs from other machines.

---

## Feature Description

### What It Does

The output stream visualization now:

1. **Current Output Display** - Shows the most recent output from the current machine at the top with orange gradient styling
2. **Scrollable History** - Displays previous outputs from the current machine in reverse chronological order below the current output
3. **Machine-Specific Filtering** - Only shows outputs generated by sequences belonging to the current machine, filtering out outputs from other machines

### User Experience

**Before**: The output stream showed ALL outputs from ALL machines, making it impossible to track which outputs came from the current machine.

**After**: The output stream shows ONLY outputs from the current machine's sequences, providing clear context and eliminating confusion.

## Implementation

### Backend Changes

#### 1. RealityEngine.ts (`/src/engine/RealityEngine.ts`)

**Modified**: `processInput()` method (lines 147-159)

**Purpose**: Tag each output with its source sequence information

**Changes Made**:
```typescript
// Before - outputs had no sequence identification
result.totalOutputs.push(...transitionResult.assertedOutputs);

// After - outputs are tagged with sequence metadata
const taggedOutputs = transitionResult.assertedOutputs.map(output => ({
  ...output,
  metadata: {
    ...(typeof output.metadata === 'object' ? output.metadata : { description: output.metadata }),
    sequenceId,
    sequenceName: sequence.name
  }
}));
result.totalOutputs.push(...taggedOutputs);
```

**How It Works**:
1. For each sequence that processes an input
2. For each output generated by that sequence
3. Add `sequenceId` and `sequenceName` to the output's metadata
4. Preserve existing metadata (description, pattern, etc.)
5. Handle both object and string metadata formats

**Result**: Every output now carries information about which sequence (and therefore which machine) produced it.

---

### Frontend Changes

#### 1. MachineContainerView.tsx (`/visualizer/frontend/src/components/MachineContainerView.tsx`)

**Modified**: Added filtering logic (lines 30-39) and updated props (line 209)

**Purpose**: Filter outputs to show only those from current machine's sequences

**Changes Made**:

**Step 1 - Add filtering logic**:
```typescript
// Get current machine's sequence IDs
const machineSequenceIds = currentMachine?.sequenceIds || [];

// Filter outputs to show only those from current machine
const filteredOutputs = currentOutputVectors.filter(output => {
  // Check if output has sequenceId in metadata
  if (output.metadata && typeof output.metadata === 'object' && 'sequenceId' in output.metadata) {
    return machineSequenceIds.includes(output.metadata.sequenceId as string);
  }
  // If no sequenceId metadata, include all outputs (backward compatibility)
  return true;
});
```

**Step 2 - Use filtered outputs**:
```typescript
// Before
<OutputStreamVisualization
  outputVectors={currentOutputVectors}
  maxVisible={10}
/>

// After
<OutputStreamVisualization
  outputVectors={filteredOutputs}
  maxVisible={10}
/>
```

**How It Works**:
1. Get the current machine's sequence IDs from the machine object
2. Filter `currentOutputVectors` array to include only outputs where:
   - Output has metadata object
   - Metadata contains `sequenceId` field
   - That `sequenceId` is in the current machine's sequence list
3. Fall back to showing all outputs if metadata is missing (backward compatibility)
4. Pass filtered outputs to OutputStreamVisualization component

**Result**: OutputStreamVisualization only receives and displays outputs from the current machine.

## Data Flow

### Complete Output Flow

1. **User applies input** → Input sent to backend API

2. **Backend processes input**:
   ```
   RealityEngine.processInput(inputVector)
   ├─ For each sequence in engine
   │  ├─ sequence.transition(inputVector)
   │  ├─ Collect assertedOutputs
   │  └─ Tag outputs with sequenceId and sequenceName
   └─ Return TransitionResult with tagged totalOutputs
   ```

3. **Backend sends WebSocket update**:
   ```
   {
     type: 'transition',
     totalOutputs: [
       {
         id: 'output-1',
         vector: [1.0, 0.0],
         timestamp: 1737734400000,
         metadata: {
           description: 'NAND output',
           sequenceId: 'seq-abc-123',      // ← NEW
           sequenceName: 'NAND(0,0) = 1'   // ← NEW
         }
       },
       ...
     ]
   }
   ```

4. **Frontend receives update**:
   ```
   WebSocket → store.setCurrentOutputVectors(outputs)
   ```

5. **MachineContainerView filters outputs**:
   ```
   currentOutputVectors (all outputs)
   ↓
   filteredOutputs = filter by currentMachine.sequenceIds
   ↓
   OutputStreamVisualization (machine-specific outputs only)
   ```

6. **OutputStreamVisualization displays**:
   ```
   Current: Latest filtered output (orange gradient)
   History: Previous filtered outputs (scrollable)
   ```

## Edge Cases Handled

### 1. Backward Compatibility
**Issue**: Existing outputs might not have sequenceId metadata
**Solution**: Filter returns `true` if no sequenceId present, showing all outputs

### 2. Missing Machine
**Issue**: No current machine selected
**Solution**: `machineSequenceIds` is empty array, filter shows all outputs

### 3. Metadata Format Variations
**Issue**: Metadata might be string or object
**Solution**: Check type and only filter when it's an object with sequenceId

### 4. Empty History
**Issue**: Only one output exists
**Solution**: Show in current section, history section hidden

### 5. Rapid Updates
**Issue**: Many outputs arriving quickly
**Solution**: React efficiently updates with filtered list, auto-scrolls history

## Update: Output History Accumulation Fix

**Date**: January 26, 2026
**Issue**: Output history was being cleared on each input instead of accumulating
**Root Cause**: Store was replacing `currentOutputVectors` instead of appending to it

### Changes Made

#### File: `/visualizer/frontend/src/store.ts`

**Issue 1 - processInput handler (line 397)**:
```typescript
// Before - REPLACED outputs
set({ currentOutputVectors: totalOutputs });

// After - APPENDS new outputs to existing history
set({ currentOutputVectors: [...get().currentOutputVectors, ...totalOutputs] });
```

**Issue 2 - WebSocket transition handler (line 765)**:
```typescript
// Before - REPLACED outputs
set({ currentOutputVectors: outputs });

// After - APPENDS new outputs to existing history
set({ currentOutputVectors: [...get().currentOutputVectors, ...outputs] });
```

**Enhancement 1 - Reset simulation (line 373)**:
```typescript
// Added clearing of output history on reset
set({
  simulationState: result.state,
  simulationProgress: 0,
  activityEvents: [],
  currentOutputVectors: []  // ← ADDED
});
```

**Enhancement 2 - Load machine (line 160)**:
```typescript
// Clear outputs when switching machines
set({
  currentMachine: machine,
  currentMachineId: machineId,
  lastViewedMachineId: machineId,
  currentView: 'administration',
  currentOutputVectors: []  // ← ADDED
});
```

### Behavior After Fix

| Action | Output History Behavior |
|--------|-------------------------|
| Apply first input | New output appears in "CURRENT" section |
| Apply second input | Previous output moves to "HISTORY", new output becomes "CURRENT" |
| Apply more inputs | All previous outputs accumulate in "HISTORY" (scrollable) |
| Reset simulation | Output history clears completely |
| Switch machines | Output history clears, new machine starts fresh |
