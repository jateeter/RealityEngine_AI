  Priority 2 — Cross-cutting Patterns

  Three systemic patterns appear across both applications:

  1. Immutable collections in hot paths — The Scala core uses Vector[Double] and List throughout the per-step path. These are safe but allocate on every mutation. The
  most impactful single change would be making PerceptualSpace mutable internally (Array[Double]) — it is the structure written to on every machine output.
  2. Full re-fetch on WebSocket events — Both store.ts and useMachineSimulation.ts respond to each step message by fetching the complete sequence graph. The WebSocket
  message should carry the delta (which vector IDs changed state) so the frontend can update in place.
  3. No blocking I/O isolation — The Scala VectorStore ties thread pool threads to network round-trips via the blocking sttp backend. In an Akka HTTP service, blocking in
   the default dispatcher degrades all request handling. This is the only fix that requires an external dependency change (async sttp backend), but it is the
  highest-impact architectural issue.
