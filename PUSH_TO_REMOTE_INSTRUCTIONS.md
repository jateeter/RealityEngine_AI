## Push Instructions

### Option 1: Standard Push (Recommended)

If the remote repository should receive your local changes:

```bash
# Push the main branch
git push origin main

# Push the v1.1.0 tag
git push origin v1.1.0
```

### Option 2: Force Push (If Remote Needs Override)

⚠️ **Warning**: Only use if you're certain local history is correct and remote changes should be discarded.

```bash
# Force push main branch (with lease for safety)
git push origin main --force-with-lease

# Push the tag
git push origin v1.1.0
```

### Option 3: Merge Remote Changes First

If remote has changes that should be integrated:

```bash
# Fetch remote changes
git fetch origin

# Rebase local changes on top of remote
git rebase origin/main

# If conflicts occur, resolve them, then:
git rebase --continue

# Push after successful rebase
git push origin main
git push origin v1.1.0
```

## Rollback Plan

If issues occur after pushing:

### Undo Tag Push
```bash
# Delete remote tag
git push origin :refs/tags/v1.1.0

# Delete local tag
git tag -d v1.1.0
```

### Undo Commit Push
```bash
# Reset local to previous state
git reset --hard HEAD~1

# Force push to remote
git push origin main --force
```
